BEGIN PROGRAM

TAKE IN THE EQUATION AS A TYPED STRING

SPLIT EVERY CHARACTER OF THE STRING INTO A LIST

CHECK FOR *INITIAL* SYNTAX REQUIREMENTS:
	-IS THERE AN EQUAL SIGN?
	-IS THERE A ZERO DIVIDE?
	-ARE THERE ONLY NUMBERS?
	-IS THERE ONLY ONE VAR?
	-ARE THERE TOO MANY VAR?
	-IS THERE A LONE MODIFIER?

ARE ALL REQUIREMENTS MET?
	YES: CONTINUE
	NO: RAISE ERROR BASED OFF OF WHICH ARE FALSE


BEGIN SOLVING PREP:
	-REGISTER ALL VALUES LEADING UP TO THE EQUALS SIGN AS ONE LIST
	-REGISTER ALL VALUES AFTER THE EQUALS SIGN AS ANOTHER LIST
	-CHECK FOR MODIFIERS NOT ATTACHED TO A NUMBER.
		-IS THE MODIFIER BEFORE A NUMBER?
			-YES: ATTACH MODIFIER TO NEXT INDEX IN LIST 
				-(only if it is a -,*, or /. If it is a +, delete it.)
			-NO: RAISE ERROR ABOUT ISOLATED MODIFIER

FOR EVERY INDEX IN BOTH LISTS, IGNORING VARIABLES AND THE NUMBERS ATTACHED TO THEM:
	-CATEGORIZE EACH NUMBER BASED OFF OF THEIR MODIFIERS.
	-MERGE LIKE-TERMS. LONE POSITIVE AND NEGATIVE NUMBERS GET MERGED.
	-MERGE LIKE-TERMS. LONE MULTIPLYING AND DIVIDING NUMBERS GET MERGED.
	-COMBINE ALL EXTRA NUMBERS NOT ATTACHED TO X 

FOR EVERY INDEX IN THE LIST WITH X ATTACHED TO THEM:
	-MERGE LIKE-TERMS. LONE VARIABLES WITH CO-EFFICIENTS GET MERGED.

FOR THE SIDE WITH X, WHILE X IS NOT ALONE:
	-FOR LONE NUMBERS: SWAP THE MODIFIER, PUT IT ON THE END OF THE OTHER SIDE.
	-FOR MULTIPLYING AND DIVIDING: SWAP THE MODIFIER, PUT IT ON THE OTHER SIDE.

ONCE X IS ALONE, GO TO THE OTHER SIDE AND, WHILE THERE IS MORE THAN 1 NUMBER ON SAID SIDE:
	-MERGE THE NUMBERS.
	-SUBTRACT NUMBERS
	-ADD NUMBERS
	-MULTIPLY NUMBERS.
	-DIVIDE NUMBERS.

IF X IS ALONE AND THERE IS ONLY ONE NUMBER ON THE OTHER SIDE:
	-MERGE THE TWO LISTS TOGETHER INTO A STRING AGAIN, DIVIDED BY AN EQUALS SIGN.
IF NOT:
	-RAISE AN ERROR BECAUSE THOSE CONDITIONS SHOULD HAVE BEEN MET.
	
END PROGRAM
